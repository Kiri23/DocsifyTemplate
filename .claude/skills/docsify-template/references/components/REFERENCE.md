# Components — Reference

Data-driven template literal components that transform YAML code fences into interactive HTML. No JSX, no bundler, no build step.

## How It Works

1. Author writes ` ```component-name ` with YAML content in markdown
2. Docsify renders it as `<pre><code class="lang-component-name">YAML</code></pre>`
3. `component-renderer.js` finds registered components in the HTML
4. Parses YAML via `js-yaml`, calls `window.ComponentName(data)`
5. Component function returns HTML string, replaces the `<pre>` block

## Current Registry

| Component | Code Fence | Data Shape | Purpose |
|-----------|-----------|------------|---------|
| CardGrid | `card-grid` | Array (unique!) | Responsive linked card grid |
| EntitySchema | `entity-schema` | Object | Expandable entity field cards |
| ApiEndpoint | `api-endpoint` | Object | REST endpoint with method badge |
| StatusFlow | `status-flow` | Object | Clickable state machine visualization |
| DirectiveTable | `directive-table` | Object | Searchable categorized reference table |
| StepType | `step-type` | Object | Workflow step card with sync/async badge |
| ConfigExample | `config-example` | Object | Annotated code with numbered callouts |
| SideBySide | `side-by-side` | Object | Two-panel layout (text, code, or component) |

**Not in registry** (used differently):
- `Tabs` — generated by component-renderer for tabbed pages
- `CodeBlock` — inline syntax highlight + copy button
- `RegionToggle` — DOM `data-region` directive processor

## Registration Checklist

A component must be registered in **two places** or it silently fails:
1. `docs/index.html` — `<script src="components/my-component.js"></script>`
2. `docs/plugins/component-renderer.js` — add `'my-component'` to `COMPONENT_REGISTRY`

Name conversion: `my-component` (kebab in markdown) → `MyComponent` (PascalCase global function).
